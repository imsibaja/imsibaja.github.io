LLM-Assisted Triage for Marine Survey Imagery

About

Modern marine wildlife surveys generate data faster than analysts can review it. A single glider deployment across the California Current can produce thousands of imagery records over two weeks — each one needing a validated species ID, a habitat tag, a data quality flag, and a field note before it can enter a species distribution model or stock assessment pipeline.

The manual triage step is the bottleneck. In this project, I build a Python pipeline that automates it using claude-sonnet-4-6 via the Anthropic API, turning raw survey metadata into structured ecological annotations at scale. This integrated approach demonstrates:

  • Structured prompting for scientific domain tasks with calibrated output constraints
  • Prompt caching to reduce API costs across large batches
  • Resilient API integration with exponential backoff and graceful error handling
  • Appending LLM annotations to an output CSV suitable for downstream ML workflows

Data

Survey Metadata Records

The pipeline accepts a CSV of pre-extracted imagery metadata — the tabular record that accompanies each image in a survey archive. A single record looks like this:

Field Value
image_id 20240318_GLDR_007
species_observed Common Dolphin
location Central CA Coast
latitude / longitude 36.49°N, 122.91°W
timestamp 2024-03-18T15:03:00
camera_type GLIDER
depth_m 22

Platform Types

Each camera_type value carries different observational constraints that the model uses when assessing data quality:

Platform Context
AERIAL_DRONE Surface-visible species only; depth = 0
GLIDER / AUV Subsurface observations; depth is reliable
ROV Precise depth; high spatial resolution
TRAIL_CAM Shoreline / haul-out / intertidal species
TOWED_CAM Depth = tow depth; variable image quality

Example Input

example_input.csv contains 15 synthetic records spanning the California Current System — blue and humpback whales, rockfish, sea otters, pinnipeds, sharks, and an unidentified towed-camera record — representative of a real multi-platform survey batch.

Build the Annotation Prompt

System prompt with prompt caching

The system prompt establishes the model’s role as a NOAA SWFSC field analyst and specifies the exact annotation schema. Marking it with cache_control tells Anthropic’s API to cache it after the first call — on batches of 50+ records this reduces input token costs by ~90%.

client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=512,
    system=[
        {
            "type": "text",
            "text": SYSTEM_PROMPT,
            "cache_control": {"type": "ephemeral"},  # cached after first call
        }
    ],
    messages=[{"role": "user", "content": build_user_prompt(record)}],
)

The system prompt instructs the model to return exactly seven fields — taxonomic classification, habitat type, behavioral context, quality assessment (text and integer), survey notes, and a confidence level — with no prose, no markdown fences, just a valid JSON object.

User prompt template

Each record is formatted into a structured request that mirrors a field log entry:

def build_user_prompt(record: dict) -> str:
    return f"""\
Analyze the following marine imagery metadata record and return a JSON object
with your ecological annotations.

METADATA:
  image_id:    {record.get('image_id', 'N/A')}
  species:     {record.get('species_observed', 'unidentified')}
  location:    {record.get('location', 'N/A')}
  latitude:    {record.get('latitude', 'N/A')}
  longitude:   {record.get('longitude', 'N/A')}
  timestamp:   {record.get('timestamp', 'N/A')}
  camera_type: {record.get('camera_type', 'N/A')}
  depth_m:     {record.get('depth_m', 'N/A')}

Return exactly this JSON structure (no other text):
{{
  "ecological_classification": "<validated species name>",
  "habitat_type": "<habitat category>",
  "behavior_notes": "<behavioral context or 'insufficient metadata'>",
  "quality_assessment": "<Excellent|Good|Marginal|Poor>",
  "quality_score": <integer 1–5>,
  "survey_notes": "<1–2 sentence scientific summary>",
  "confidence_level": "<High|Medium|Low>"
}}"""

Call the API

Retry logic

The API layer wraps each call in an exponential backoff loop. Rate-limit responses include a retry-after header that gives a more accurate wait time than a fixed formula.

for attempt in range(MAX_RETRIES):
    try:
        response = client.messages.create(...)
        return json.loads(response.content[0].text.strip())

    except anthropic.RateLimitError as exc:
        retry_after = float(
            exc.response.headers.get("retry-after", BASE_RETRY_DELAY * (2 ** attempt))
        )
        time.sleep(retry_after + random.uniform(0, 1))

    except anthropic.APIStatusError as exc:
        if exc.status_code >= 500:
            time.sleep(BASE_RETRY_DELAY * (2 ** attempt) + random.uniform(0, 1))
        else:
            raise  # 4xx errors are not retryable

Parse and write output

Records that return valid JSON are merged with their original metadata and written to the output CSV. Records that exhaust all retries are written with ecological_classification = "ERROR" and quality_score = 1, keeping the output file complete so downstream steps can filter rather than crash.

Reflection

The pipeline successfully annotated all 15 records with ecologically plausible outputs. The model’s reasoning was strongest where metadata was richest — ROV records with precise depth and geolocation consistently returned quality scores of 5 and detailed survey notes. Glider-based cetacean records were correctly rated Marginal, as glider imagery has known limitations for species-level ID at depth.

A few limitations worth noting. The pipeline annotates metadata, not images — it can’t detect blur, backscatter, or poor framing that only a human reviewer or CV model would catch. The quality scoring therefore reflects metadata completeness and ecological plausibility rather than actual image quality. Additionally, the current system prompt is below the 2,048-token threshold needed to activate prompt caching on Sonnet 4.6; embedding a species taxonomy table or survey SOP would push it past threshold and cut costs substantially on larger batches.

The natural next step for production scale is the Anthropic Batches API — submit an entire survey batch as a single job, poll for completion, write results in one pass, and cut per-token costs by 50%.
